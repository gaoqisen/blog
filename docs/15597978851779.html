一、证照数据处理

名词解释

  • 重复数据删除
    • 通过获取同一个证照编码的条数大于2（结婚离婚证）或者1（非结婚离婚证）的证照编码。删除其他数据仅仅保留1或2条数据（已上传[ex_state=1]的不删除）。
    • 查看证照编码重复数据sql： SELECT CERTIFICATE_NUMBER FROM ( SELECT COUNT(CERTIFICATE_NUMBER) num, CERTIFICATE_NUMBER FROM LOUSHANGFPE.LIC_DATA_EX where CERTIFICATE_NAME in (‘证照名称’) AND EX_STATE <> ‘4’ GROUP BY CERTIFICATE_NUMBER HAVING count(CERTIFICATE_NUMBER) > ？ ORDER BY num DESC)
    • 非结婚离婚证：重复数据有ex_state=1的， 删除ex_state=2的数据。否则保留一条数据，删除其他ex_state=2的数据。

  • 时间格式处理：将 《31-10月-05 12.00.00.000000000 上午》 格式的时间刷新为 《yyyy-MM-dd》格式。

    • 查看时间异常sql: SELECT * FROM LIC_DATA_EX WHERE EX_STATE = ‘2’ and length( ISSUE_DATE ) <> lengthb( ISSUE_DATE ) and ISSUE_DATE is not null。 并刷新时间。

  • 部门信息找回：

    • 通过部门名称和指定的区县的编码等刷新部门编码以及部门名称，部门信息在loushagnpfe的DEPT_INFO_FULL里面。
    • 生成执行sql如：UPDATE LIC_DATA_EX SET ISSUE_DEPT = ‘重庆市九龙坡区民政局’, ISSUE_DEPT_CODE = ‘11500107009293137E’,CERTIFICATE_AREA_CODE = ‘500107000000’ where ISSUE_DEPT LIKE ‘%九龙坡%’ AND CERTIFICATE_NAME in (‘结婚证’, ‘离婚证’, ‘结婚证补办’, ‘离婚证补办’) AND ex_state = 2

  • 证书信息找回：

    • 通过证书信息批量更新证书信息。证书信息在loushagnpfe的CERTIFICATE_INFO_FULL。
    • 生成执行sql如：UPDATE LIC_DATA_EX SET CERTIFICATE_TYPE = ‘中华人民共和国结婚证’,CERTIFICATE_TYPE_CODE = ‘中华人民共和国结婚证’, CERTIFICATE_HOLDER_CODE = ‘111’, CERTIFICATE_NAME_CODE = ‘5000000010-20190125173606’ where CERTIFICATE_NAME = ‘结婚证’ AND ex_state = 2

  • 找回部门名称（只是针对结婚离婚证照, 暂时弃用）

    • 证照数据部门名称：证照系统处理数据之后，部门名称很多都是重庆市民政局，是不正确的。通过证照编码在124数据源里面找回证照数据的部门名称。

  • 证书名称：用户选择证书名称，处理相应的数据。

  • SQL文本：用户填写限制条件，通过限制条件处理数据。

  • 数据状态：lic_data_ex表里面的ex_state指定的状态。1 已上传、2 待处理、8 结婚离婚证待上传(数据处理后没问题的)、7 食品许可证待上传(数据处理后没问题的)

操作流程

  • 需要按照步骤依次处理：时间格式处理 > 重复数据删除 > 部门信息找回 > 证照信息找回

注意事项

  • 尽量按照步骤执行处理数据。

二、证照数据上报

将处理正常的数据批量推送到国家中间库。并在对账表里面添加记录与更改推送过的数据状态。

名词解释

  • 证书名称：用户选择证书名称，处理相应的数据。

  • SQL文本：用户填写限制条件，通过限制条件处理数据。

  • 数据状态：lic_data_ex表里面的ex_state指定的状态。1 已上传、2 待处理、8 待上传(数据处理后没问题的)

  • 推送数量：填写推送数据，进行指定数量的数据推送。

操作流程

用户填写证书名称、SQL文本（选填）、数据状态、推送数量。

注意事项

  • 注意推送是的数据状态，需要选择待上传选项。待上传ex_state=8、待处理ex_state=2

  • 在点击推送数据至国家中间库之后，会返回一条正常的数据， 用户确认数据无误之后，点击确认按钮进行推送。

三、用户数据处理

通过用户名、身份证号码、手机号码推送用户数据到用户库，密码为身份证号码后6位。

名词解释

  • 表名： 将用户数据存放在新建的一张表（身份证号码、手机号、用户名、状态、ID）里面。通过表里面的数据进行上传。

  • 统一身份认证数据推送：点击后将填写表里面的数据推送到用户数据库。

  • 用户重复删除：由于推送是用多线程批量处理的数据，线程数量为10， 每批推送1000条。 如果这10批数据每批里面都有重复的数据，那可能会出现重复数据上传。该按钮保证数据不重复。

  • 导入数据类型: 导入2批数据， 一批没有手机号码（REGISTER_TYPE＝4），一批有是手机号码（REGISTER_TYPE＝3）。

每隔5秒执行一次：”/5 ?”
每隔1分钟执行一次：”0 /1 ?”
每天23点执行一次：”0 0 23 ?”
每天凌晨1点执行一次：”0 0 1 ?”
每月1号凌晨1点执行一次：”0 0 1 1 ?”
每月最后一天23点执行一次：”0 0 23 L ?”
每周星期天凌晨1点实行一次：”0 0 1 ? L”
在26分、29分、33分执行一次：”0 26,29,33 ?”
每天的0点、13点、18点、21点都执行一次：”0 0 0,13,18,21 ?”
表示在每月的1日的凌晨2点调度任务：”0 0 2 1 ? “
表示周一到周五每天上午10：15执行作业：”0 15 10 ? * MON-FRI”
表示2002-2006年的每个月的最后一个星期五上午10:15执行：”0 15 10 ? 6L 2002-2006”
`

  • 删除：删除一个定时任务，如果定时任务在执行，则先停止任务后在删除任务。

  • 修改：修改定时任务参数。 如果定时任务已经运行，则需要重启定时任务。

  • 开始/停止：点击开始按钮后，按钮会变为停止。点击停止按钮后，按钮会变为开始。

操作流程

点击创建定时任务后填写相应表单数据后，点击确认则创建了一个定时任务，默认未启动，如果需要启动定时任务则点击开始按钮。

注意事项

  • Crontab表达式必须填写正确

  • 任务名称尽量详细的表述定时任务的功能、便于区分和了解定时任务的作用。

  • 系统启动后，会初始化定时任务， 如果需要启动后不执行定时任务，需要先停止定制任务。

六、数据源

添加多个数据源，便于后期更改推送数据。

名词解释

  • 别名：用于区分多个数据源的名称。建议：Mysql_db_ipAddress。
  • URL：连接数据库的链接包含IP和参数。如：jdbc:mysql://192.168.0.100:3306/shgb_fz?useUnicode=true&characterEncoding=UTF8&autoReconnect=true&zeroDateTimeBehavior=convertToNull
  • username: 连接用户名。
  • password: 对应用户名的密码。

操作流程

数据填写完之后点击保存即可。

注意事项

  • url填写详细。
  • 用户名、密码必须正确。
  • 文章目录
  • 站点概览
